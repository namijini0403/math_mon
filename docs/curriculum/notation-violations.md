# 풀이 표기 위반 추적 (curriculumGuard가 발견) — 순차 정비 대기 목록

> 2026-06-14 가드 1차 스캔(스킬당 25시드)에서 발견된 **학년 부적합 표기**. 단원 audit 시 고치고, 고치면 줄에 ✅.
> 가드(client/src/game/curriculumGuard.ts)는 AUDITED_UNITS에 등록된 단원만 실패로 강제 — 아래는 미등록(=아직 도달 전) 단원의 알려진 위반.

## 1학년 (× 곱셈기호 — 곱셈은 2학년)
- [ ] `num50-compose` (1-1 50까지의 수): 풀이에 `×` 사용 → "10개씩 N묶음과 낱개 M개" 말로.
- [ ] `num100-compose` (1-2 100까지의 수): 풀이에 `×` 사용 → 자리값 말로(몇십 = 10개씩 묶음).

## 2학년 (÷ 나눗셈기호 — 나눗셈은 3학년)
- [ ] `muli-times` (2-1 곱셈): 풀이에 `÷` → 곱셈 말("몇씩 몇 묶음")로.
- [ ] `gugu-missing` (2-2 곱셈구구): `□`를 `÷`로 구함 → "N단을 외워 보면 N×?=…" 식으로.
- [ ] `len22-conv` (2-2 길이 재기): 단위 변환에 `÷` → "100 cm = 1 m" 묶음 말로.
- [ ] `time2-conv`, `time2-units` (2-2 시각과 시간): `÷` → "60분 = 1시간" 말로.
- [ ] `rule2-shape`, `rule2-stack`, `rule2-word` (2-2 규칙 찾기): `÷`/곱셈 표기 점검.

## 6학년 (숫자 거듭제곱 2² — 지수는 중등, 초등은 `n × n`)
- [ ] `ch61-cube-edge` (6-1 심화): `n²` → `n × n`.
- [ ] `ch62-circle-biggest`, `ch62-compound-area`, `ch62-cylinder-net`, `ch62-quarter-cut`, `ch62-rotate-cross-diff` (6-2 심화): 원 넓이를 `r² × 3.14` → `r × r × 3.14`(초등 원의 넓이 = 반지름 × 반지름 × 원주율).

> 참고: 정식(비심화) 6학년 원의 넓이 단원(unitCircle)도 도달 시 같은 `r × r × 원주율` 표기 확인.
> 부등호(<,>)는 앱 결정상 사용 OK(입력은 클릭 선택만) — 가드 대상 아님(docs/curriculum/g1.md 참조).
