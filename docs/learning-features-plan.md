## 0. 현재 기반 (재사용)
- `store.skillStats: Record<skillId,{c,w}>` — 유형별 정오답 누적(서버 `progress.skill_stats` 동기화, 교사창 노출).
- `store.recentWrong: string[]` — 최근 틀린 skillId(최대 8, 휘발적).
- 생성기 각 문제에 `explanation: MathExpr`(풀이 토큰) 이미 존재 → 풀이 기능의 출발점.
- 문제는 시드 기반 파라메트릭(`RNG(seed)`) → 틀린 문제 '그대로 다시보기'는 (skillId, seed)만 저장하면 재현 가능.
- 단원↔스킬: `SkillDef.unitId`, 단원↔학기: gradeAccess. 연습/시험: PracticePage(/practice/:mode/:unitId), ExamPage(/exam/:unitId).

## 1. 틀린 문제 다시보기 + 같은 유형 무한 연습
**요구**: 단원별로 틀린 문제만 다시 보는 창(게임 세계관 이름, '오답노트' 금지). 틀린 것과 **같은 유형 무한 반복** 연습. 단원평가 연습 창에도 "여러 유형 섞임(현행)"과 별도로 **"내가 틀린 유형 중심"** 모드(이름도 멋지게).

**이름 후보(밤하늘 왕국·봉인 세계관)**:
- 틀린 문제 다시보기 = **「흐려진 별의 회랑」** (틀린 문제 = 빛을 잃은 별, 다시 풀면 별이 켜진다)
- 같은 유형 무한 수련 = **「수련의 첨탑」** 또는 **「무한 단련장」**
- 시험 약점 중심 모드 = **「약점 봉인 모드」** / **「그림자 정복」**
(최종 명칭은 구현 시 확정)

**설계**:
- 틀린 문제 저장: `wrongLog: Record<unitId, Array<{ skillId; seed; ts }>>`(최근 N개, 중복 시 갱신). recordAnswer 확장 — 오답 시 push, 정답 시 같은 (skillId,seed) 제거.
- 회랑 화면: 단원 선택 → 저장된 (skillId,seed)로 **그 문제 그대로 재현**. 맞히면 별 점등(목록에서 제거).
- 무한 수련: skillId 1개로 `generateProblem(skill, RNG(randomSeed()))` 무한 출제(PracticePage 변형 — unit 풀 대신 단일 skill 풀).
- 시험 약점 모드: ExamPage에서 풀을 "오답 많은 skill 가중"으로 구성(skillStats의 w 비율↑ 우선).

**검증**: wrongLog reducer 순수화+단위테스트, 재현성(같은 seed=같은 문제) 테스트.

## 2. 힌트권 / 보스 힌트 + 연출
**요구**: 특정 아이템 획득 시 힌트권 부여. 또는 특정 보스 봉인 시 그 보스가 사용자가 고른 2~3문제 힌트 제공. 연출 그럴듯하게.

**설계**:
- `hintTokens: number` 인벤토리. 가챠/보스 보상으로 적립(특정 아이템 = 힌트권 묶음).
- 문제 화면에 "힌트 보기(🔮)" 버튼 — 토큰 1 소모 → 단계적 힌트(풀이의 첫 스텝만 공개 → 더 보기). 풀이(기능4)와 공유.
- 보스 힌트 연출: 봉인한 보스가 말풍선으로 힌트를 건네는 컷(보스 일러스트 + 반짝임). "2~3문제 힌트권" = 토큰 N + 어느 문제에 쓸지 선택.
- 힌트는 정답 직답 금지(첫 단계/접근법만). 남용 방지 = 토큰 소모.

**검증**: 토큰 적립/소모 순수 로직 테스트. 힌트 단계가 정답을 바로 노출하지 않는지 점검.

## 3. 교사창 — 약점 유형 정리 + 인쇄 + 교사 발송 문제
**요구**: 학생별 어려워하는 유형 정리(다 어려우면 쉬운 유형부터). 그 유형만 모아 **인쇄** 가능. 교사가 보내는 문제(5~10문제)를 특별 미션처럼 푸는 기능.

**설계**:
- 교사창: 학생별 skill_stats → 약점 유형 정렬(정답률 낮은 순; 표본 적으면 난이도 낮은 유형부터). 유형명·정답률·표본수.
- 인쇄: 선택 유형들의 문제를 N개 생성해 **인쇄용 정적 페이지**(문제+정답 분리, 시드 표기). 브라우저 인쇄.
- 교사 발송 문제 = 기능5의 '특별 미션'과 통합(아래). 5~10문제 세트를 학생에게 푸시 → 학생 홈에 "특별 미션 도착" 알림.

**검증**: 약점 정렬 로직 순수 테스트. 인쇄 페이지는 시각 확인(Playwright).

## 4. 풀이(해설) 보기 — 교육과정 정확성 최우선 ⚠️
**요구**: 답 제출 시 정오만이 아니라 **풀이**도 볼 수 있게. **아이 눈높이 + 교육과정 용어·내용**. (예: 최소공배수에 제곱 표기 금지 등.)

**설계**:
- 각 문제 `explanation`(이미 존재) → 단계별 풀이 뷰로 노출(피드백 시트 "풀이 보기" 토글).
- **유형별 풀이 템플릿**: 단순 답이 아니라 "왜/어떻게"를 학년 수준 언어로. 생성기별로 풀이 스텝을 점검·보강.
- **교육과정 가드(절대 준수)**: 학년별 허용 표기·용어만. 의심 시 **2022 개정 교육과정 성취기준 검색 후 작성**. 예시 금지: 지수(제곱), 미지수 x(저학년), 음수(해당 학년 전), 소인수분해 표기(초등 X) 등.
- 단계 점진 공개(힌트와 공유): 1단계 → 2단계 → 정답.

**리스크/방법**: 스킬 수백 개 → 포맷·단원 단위로 나눠 점검. 각 학년 풀이 작성 전 성취기준·교과서 용어 확인. **풀이 텍스트 교육과정 적합성 자체 점검 + 필요 시 사용자 확인.**

**검증**: 풀이 토큰 렌더 스냅샷, 금지 표기(제곱^·x·음수 등) 정규식 스캔 테스트(학년 가드), 표본 수동 검수.

## 5. 단원평가 모드 (게임과 분리 · 교사 배포형) ⭐ 대형
**요구**:
- 평소엔 잠겨 있다가 **교사가 교사창에서 배포하면 갑자기 도착**(특별 시험처럼).
- 배포 옵션:
  - **우리 반이 많이 틀린 유형 중심** 출제.
  - 기본 난이도 **상/중/하**: 하=계산 중심, 중=문장제 중심, 상=심화 중심(단, '하'라고 계산만 잔뜩 X — **비율·문항수 모두 교사 조절 가능**).
  - **학생마다 다른 시험** 배포 가능(그 학생 틀린 유형 중심).
- 교사 확인 항목:
  - 학생이 **어떤 문제를 틀렸는지**, **몇 점**인지.
  - **어떤 난이도**의 시험을 봤는지.
  - **점수 변화 추이**(해당 단원뿐 아니라 **현재까지 단원평가 누적**).
  - **강·약 영역 간단 레포트**.

**설계(서버+교사UI+학생UI+분석)**:
- 서버: `assignments`(교사 발행: classCode/studentId·unitId·구성(난이도 비율·문항수·약점가중)·발행시각), `assignment_results`(student·assignment·score·문항별 정오(skillId·correct)·제출시각). 인증=교사키/JWT.
- 학생: 홈에 "📜 도착한 시험" — 잠금 해제식 진입. 제출 시 결과 서버 기록.
- 교사창: 시험 작성기(난이도 상/중/하 슬라이더 = 계산/문장제/심화 비율 + 문항수), 대상(반 전체/개별/약점기반), 발행. 결과 보드(학생별 점수·문항별 오답·난이도·**누적 추이 그래프**·강약 레포트).
- 출제 엔진: 난이도 비율 → 스킬 풀에서 계산형/문장제형/심화형 선택(스킬에 형 분류 필요 — difficulty/format 메타). 약점가중 = 반/개인 skill_stats 반영.
- 누적 추이: assignment_results 시계열 → 단원평가 점수 변화.

**검증**: 출제 비율 분배 순수 테스트(상/중/하 비율→문항 구성), 약점가중 분포 테스트, 서버 라우트 tsc, 교사/학생 흐름 Playwright.

## 권장 실행 순서 (단계)
1. **Phase A (기반+기능1)**: wrongLog 모델 + 「흐려진 별의 회랑」 + 「수련의 첨탑」 + 시험 약점 모드. (클라, 재현성 테스트)
2. **Phase B (기능4 풀이)**: explanation→단계 풀이 뷰 + 학년 가드 스캔 테스트. **교육과정 검색 동반.** (단원/포맷 단위로 점진)
3. **Phase C (기능2 힌트)**: hintTokens + 힌트 단계(풀이 공유) + 보스 힌트 연출.
4. **Phase D (기능3+5 교사·단원평가)**: 서버 assignments/results + 교사 작성·배포·레포트 + 학생 수신/응시 + 누적 추이 + 인쇄. (가장 큼, 서버 중심)

각 Phase: tsc(client+server)→vitest(신규 순수테스트 포함)→build→Playwright 시각 검수→커밋·푸시. 교육과정 의심 시 검색·사용자 확인.

## 교육과정 가드 메모(작성 중 계속 보강)
- 초등 금지 표기/개념: 지수(제곱·세제곱), 미지수 문자 x·y(고학년 '□'·'어떤 수'는 OK), 음수, 무리수, 소인수분해 표기(2×2×3 나열은 OK, 2²×3 금지), 분수 나눗셈은 6학년, 비·비율 6학년, 약수·배수·약분·통분 5학년 등 — 단원 학년 확인 후 풀이 작성.
