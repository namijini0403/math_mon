# 2-1 심화 유형 구현 명세 (최고수준수학 2·3단계 유형화 — 원문 미사용, 구조만)

> 출처 유형만 추출. **숫자·소재·표현 전부 새로**(원문 복제 금지). 4단계(Top) 유형 제외.
> 공통: 모든 스킬 `challenge: true`, `difficulty: 3`. format은 fill-blanks 또는 choice.
> 스킬 id 규칙: `ch21-{slug}`. unitId는 해당 단원 id.

## ⚠️ 2학년 교육과정·표기 제약 (반드시 준수)
- **부등호 기호(<, >) 금지** → 해설·문제 모두 "보다 큰/작은", "더 많은/적은" 등 **말로**.
- **나눗셈 기호(÷) 금지**(나눗셈은 3학년) → "몇씩 몇 묶음", "□묶음", "□×6=42에서 □" 식으로 우회.
- 곱셈기호 × 는 OK(2학년 도입). 단 곱셈은 "몇씩 몇 묶음/몇 배" 반복덧셈 맥락. 곱은 한 자리×한 자리 위주.
- **fill-blanks 답에 0 금지**(양의 정수). 복수 빈칸은 blankAnswers 배열.
- 그림 필수 유형(칠교판·쌓기나무·점판·한붓그리기·종이접기 도해)은 제외. 분류는 **이모지/낱말 목록**으로 텍스트 표현.
- §2 재시도 패턴(폴백 let + for(tries<N) + 성공 시 일괄 대입 후 break) 준수. RNG(seed)만.

## unitNum3d (세 자리 수) — `ch21-card`, `ch21-coin`, `ch21-cond`

### `ch21-card` 숫자 카드로 세 자리 수 만들기 (몇 번째 큰/작은 수)
- 서로 다른 숫자 카드 4장(0 포함 가능, 1~9 중 3장 + 0 섞기) 중 **3장**으로 세 자리 수.
- "두 번째로 작은 수" 또는 "두 번째로 큰 수" 또는 "세 번째로 작은 수" 랜덤.
- 생성: 4장 카드에서 3-순열 전부 만들되 **백의 자리 0 제외**, 중복 제거·정렬 → n번째 선택.
  카드는 모두 다른 숫자(중복 없음)로 골라 순열이 충분(≥6개)하도록.
- fill-blanks 답 = 세 자리 수. 해설: 자리값 크기 비교(0은 백의 자리 불가) 후 나열.

### `ch21-coin` 동전 금액 / 몇 개 더 필요
- 두 변형 랜덤:
  (a) **합산**: "500원 a개, 100원 b개, 50원 c개, 10원 d개면 모두 얼마?" → 합(원). a~d는 0~6, 합이 세 자리 수 되게.
  (b) **부족분**: "목표 T원이 되려면 50원(또는 100원)짜리 동전이 몇 개 더 있어야?" 현재 가진 금액 cur < T, (T−cur)가 단위의 배수 → 답 = (T−cur)/단위. 0 금지(≥1).
- fill-blanks. 해설: 단위별 금액 환산 합.

### `ch21-cond` 세 조건을 만족하는 세 자리 수
- 조건 3개: ① A보다 크고 B보다 작다(말로, A<B 범위 80~120 폭) ② 십의 자리 숫자 = 일의 자리 숫자 ③ 십의 자리 숫자 = 백의 자리 숫자 + k(k=1~5).
- 생성: 범위·k·관계로 후보를 전수 생성, **해가 정확히 1개**가 되는 (A,B,k) 조합만 출제(폴백 보유).
- fill-blanks 답 = 그 세 자리 수. 해설: 조건 차례로 적용.

## unitFigure2 (여러 가지 도형) — `ch22-side`, `ch22-square-row`, `ch22-fold`

### `ch22-side` 다각형 변·꼭짓점 수 사칙 조합
- 도형별 변=꼭짓점 수: 삼각형3·사각형4·오각형5·육각형6.
- "㉠ (도형)의 변의 수 + ㉡ (도형)의 꼭짓점의 수 − ㉢ (도형)의 변의 수" 형태(랜덤 3도형, 변/꼭짓점 랜덤).
- 결과가 **양의 정수**가 되도록 조합 선택(− 항이 너무 크지 않게). fill-blanks 답 = 결과.
- 해설: 각 도형 변/꼭짓점 수 제시 후 계산. (2학년: 변·꼭짓점 수 학습 범위.)

### `ch22-square-row` 한 줄로 이어 붙인 정사각형에서 사각형 개수
- "정사각형 n개를 한 줄로 이어 붙였을 때 찾을 수 있는 크고 작은 사각형은 모두 몇 개?" n=3~6.
- 답 = n + (n−1) + … + 1 = n(n+1)/2. fill-blanks.
- 해설: 1칸짜리 n개, 2칸짜리 (n−1)개, … 합. (그림 없이 말로 서술 — minVariety: 4.)

### `ch22-fold` 색종이 접어 자르기 (사각형 개수)
- "색종이를 같은 방향으로 n번 접었다 펼친 뒤 접힌 선을 따라 모두 자르면 사각형이 몇 개?" n=2~4.
- 답 = 2ⁿ (2,3,4 → 4,8,16). fill-blanks. 해설: 한 번 접을 때마다 칸 수 2배. minVariety: 3.

## unitAddSub2 (덧셈과 뺄셈) — `ch23-between`, `ch23-vertical`, `ch23-symbol`

### `ch23-between` 두 식 사이 두 자리 수의 개수
- "(A+B)보다 크고 (C−D)보다 작은 두 자리 수는 모두 몇 개?" 좌=A+B, 우=C−D, 좌<우, 둘 다 두 자리.
- 답 = (우−1)−(좌+1)+1 = 우−좌−1, 2~9개 되게 구성. fill-blanks. 부등호 **말로**.
- 해설: 좌·우 계산 후 사이 수 나열/개수.

### `ch23-vertical` 세로셈 빈칸 복원 (받아내림 포함 두 자리 뺄셈/덧셈)
- 두 자리 ± 두 자리, 일부 자리 숫자를 ㉠㉡로 가리고 복원. 받아내림/올림이 있는 경우 포함.
- 생성: 완전한 식 먼저 만들고(받아내림 발생) 두 자리(㉠,㉡)를 가린다. 각 자리 0 아님.
- fill-blanks 2칸 [㉠, ㉡]. 해설: 일의 자리부터 받아내림/올림 추적.

### `ch23-symbol` 기호 연립 (대입으로 값 구하기)
- "▲+●+●=S, ◆−▲=D, ●=p일 때 ◆는?" 또는 3식 연쇄.
- 생성: ●=p → ▲=S−2p>0 → ◆=▲+D. 모든 중간값 양의 정수·두 자리 이내 보장.
- fill-blanks 답 = ◆. 해설: 대입 순서대로.

## unitLength2 (길이 재기) — `ch24-iterate`, `ch24-diff`, `ch24-wire`

### `ch24-iterate` 단위 길이 환산 (몇 번)
- "A 1개의 길이가 B로 m번입니다. A k개의 길이는 B로 몇 번인가요?" 답 = k×m. (m,k=2~9, 곱 ≤ 81.)
- 변형: "A 1개 = B로 m번, A의 길이가 ?개면 B로 몇 번" 단순 곱. fill-blanks.
- 해설: 1개가 m번 → k개는 m을 k번 더함(=k×m).

### `ch24-diff` 단위 반복 길이의 차
- "한 걸음이 a cm인 사람이 n걸음, 한 걸음 b cm인 사람이 n걸음. 두 길이의 차는?" 답 = n×(a−b), a>b.
- 또는 "1 cm가 ㉮ p번, ㉯ q번 → 길이의 차"=p−q. 랜덤. fill-blanks cm. 양수 보장.
- 해설: 각 길이 = n×걸음(또는 1cm 횟수), 차.

### `ch24-wire` 철사로 직사각형 만들기 (네 변 합)
- "철사를 구부려 가로 a cm, 세로 b cm인 직사각형을 만들었습니다. 사용한 철사는 몇 cm?" 답 = a+b+a+b.
- (÷·둘레공식 금지 — 네 변을 더하는 말로.) a,b=2~12. fill-blanks cm.
- 해설: 네 변 길이 = a, b, a, b → 모두 더하기.

## unitClassify (분류하기) — `ch25-count-diff`, `ch25-two-attr`, `ch25-relation`

> 분류 대상은 **이모지/낱말 목록**으로 prompt에 텍스트 표현(그림 대체).

### `ch25-count-diff` 분류 후 최다−최소 개수의 차
- 12~16개의 항목(예: 과일/곤충/탈것 이모지)을 3~4종류 중에서 무작위 배치한 목록 제시.
- "가장 많은 것과 가장 적은 것의 수의 차는?" 답 = max−min(≥1). fill-blanks.
- 해설: 종류별 개수 센 표(텍스트) 제시 후 차.

### `ch25-two-attr` 두 기준 동시 만족 개수
- 항목마다 (모양, 색) 두 속성을 가진 목록(이모지+낱말로). "○색이면서 △모양인 것은 몇 개?"
- 답 = 두 조건 모두 만족 개수(≥1 보장). fill-blanks. 해설: 두 기준 교차 표.

### `ch25-relation` 분류표 관계식으로 미지수 구하기
- "다리 없음 x마리, 2개 y마리, 4개 z마리. 전체 N마리이고 ○가 △보다 k 더 많을(적을) 때 □는?"
- 생성: 정수해 1개 되도록 역산. fill-blanks. 해설: 합·차 관계로 계산. 부등호 없이 "더 많은/적은"으로.

## unitMulIntro (곱셈) — `ch26-array`, `ch26-leftover`, `ch26-combo`

### `ch26-array` 배열 곱셈 / 묶음 역산
- 두 변형 랜덤:
  (a) "한 상자에 r개씩 c줄, b상자면 모두 몇 개?" 답 = r×c×b (각 ≤6, 곱 적당).
  (b) **역산**: "한 악기에 줄이 g개. 전체 줄이 T개일 때 악기는 몇 대?" g×□=T → □=T/g(나눗셈 기호 없이 "g씩 묶으면 몇 묶음"). 답 ≥2.
- fill-blanks. 해설: 반복덧셈·묶음으로.

### `ch26-leftover` 만들고 나눠 주고 남은 수
- "한 봉지에 p개씩 q봉지 있는 사탕을, r개씩 s명에게 주었습니다. 남은 사탕은?" 답 = p×q − r×s > 0.
- (p,q,r,s 작게, 두 곱 모두 ≤ 두 자리.) fill-blanks. 해설: 전체 = p×q, 준 것 = r×s, 차.

### `ch26-combo` 옷 조합 (몇 가지 방법)
- "윗옷 m가지, 치마 n가지를 짝지어 입는 방법은 모두 몇 가지?" 답 = m×n. (m,n=2~6.)
- fill-blanks. 해설: 윗옷 1가지마다 치마 n가지 → m묶음.
