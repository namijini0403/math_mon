## 1. 수의 범위와 어림하기

### 유형 1-1: 올림·버림 적용 후 차 구하기
- 단계: 2단계
- 구조: 어떤 수 N을 [올림/버림]하여 [단위 자리]까지 나타낸 값과 [올림/버림]하여 [다른 단위 자리]까지 나타낸 값의 차를 구하시오.
- 파라미터:
  - `N`: 5~6자리 자연수 (예: 30000~99999)
  - `round_up_place`: 올림 적용 자릿수 {천, 만}
  - `round_down_place`: 버림 적용 자릿수 {백, 천}
  - 제약: 두 근삿값이 다르게 나오도록 N의 해당 자릿수 ≠ 0
- 예시(변형 수치): "어떤 수를 올림하여 만의 자리까지 나타내면 50000이고, 버림하여 천의 자리까지 나타내면 43000입니다. 두 값의 차는 얼마입니까?"
- 풀이 절차:
  1. N의 조건 범위 설정 (올림 결과 역산)
  2. 버림 결과 역산
  3. 두 값 차 계산
- 오답 포인트: 올림·버림 방향 혼동, 자릿값 단위 오독
- 답 형식: 자연수(fill-blanks)

### 유형 1-2: 올림·버림을 이용한 상거래 최솟값 문제
- 단계: 2단계
- 구조: 물건값 P원을 지불하기 위해 A는 [단위_a]짜리 지폐만, B는 [단위_b]짜리 지폐만 사용합니다. 각자가 최소로 내야 하는 장수의 차는 몇 장입니까?
- 파라미터:
  - `P`: 구입 가격 (예: 10000~30000, 끝자리 불규칙)
  - `unit_a`: A가 쓰는 지폐 단위 {10000, 5000}
  - `unit_b`: B가 쓰는 지폐 단위 {1000, 500}
  - 제약: P가 unit_a와 unit_b 모두의 배수가 아니어야 함
- 예시(변형 수치): "14700원짜리 물건을 사려 할 때, A는 10000원짜리 지폐만, B는 1000원짜리 지폐만 사용합니다. 두 사람이 최소로 내야 하는 지폐 수의 차는 몇 장입니까?"
- 풀이 절차:
  1. A: P를 unit_a로 올림 → 필요 장수 계산
  2. B: P를 unit_b로 올림 → 필요 장수 계산
  3. 두 장수의 차 계산
- 오답 포인트: 거스름돈 계산으로 오해, '최소'를 '최대'로 해석
- 답 형식: 자연수(fill-blanks)

### 유형 1-3: 버림 적용 묶음 판매 수익 계산
- 단계: 2단계
- 구조: 물건 N개를 [묶음_a]개씩 묶어 [가격_a]원에 팔고, 남은 것은 [묶음_b]개씩 묶어 [가격_b]원에 모두 팔았을 때 총 금액은?
- 파라미터:
  - `total_items`: 총 개수 (예: 1000~9999)
  - `bundle_a`: 큰 묶음 단위 (예: 100)
  - `price_a`: 큰 묶음 가격
  - `bundle_b`: 작은 묶음 단위 (예: 10)
  - `price_b`: 작은 묶음 가격
  - 제약: `total_items`를 `bundle_a`로 나눈 나머지가 0이 아니어야 하고, 그 나머지를 `bundle_b`로 나눈 나머지도 0이 아니어야 함
- 예시(변형 수치): "과자 3847개를 100개씩 묶어 8000원에 팔고, 나머지는 10개씩 묶어 500원에 모두 팔았습니다. 과자를 판 금액은 모두 얼마입니까?"
- 풀이 절차:
  1. 큰 묶음 수 = total_items ÷ bundle_a (버림) → 큰 묶음 금액
  2. 나머지 = total_items mod bundle_a → 작은 묶음 수 = 나머지 ÷ bundle_b (버림)
  3. 총 금액 = 큰 묶음 금액 + 작은 묶음 금액
- 오답 포인트: 남은 개수 계산 오류, 묶음 단위 버림 적용 누락
- 답 형식: 자연수(fill-blanks)

### 유형 1-4: 수 카드로 조건 만족하는 수 열거
- 단계: 2단계
- 구조: [n]장의 수 카드 중 [k]장을 뽑아 [자릿수]짜리 수를 만들 때, 반올림하여 [기준 자리]까지 나타내면 [목표값]이 되는 수를 모두 쓰시오.
- 파라미터:
  - `cards`: 수 카드 집합 (0~9 중 5개, 예: {2,3,5,7,9})
  - `pick_k`: 뽑는 카드 수 (3)
  - `round_place`: 반올림 적용 자릿수 {십, 백, 천}
  - `target`: 반올림 후 목표값
  - 제약: 카드 중복 사용 불가, 첫 자리 0 사용 불가
- 예시(변형 수치): "수 카드 {2, 4, 6, 8, 3} 중 3장을 뽑아 세 자리 수를 만들 때, 반올림하여 백의 자리까지 나타내면 400이 되는 수를 모두 쓰시오."
- 풀이 절차:
  1. 반올림 후 목표가 되는 범위 설정 (예: 350 이상 450 미만)
  2. 카드 조합으로 범위 내 수 열거
  3. 조건 불만족 수 제거
- 오답 포인트: 범위 경계값 포함/미포함 오류, 카드 중복 허용 실수
- 답 형식: fill-blanks (여러 수 나열)

### 유형 1-5: 범위 조건을 이상·이하·초과·미만으로 표현
- 단계: 2단계
- 구조: □에 들어갈 수 있는 수의 범위를 이상, 이하, 초과, 미만 중 하나를 사용하여 나타내시오. 부등식: □ + A > B 또는 □ × A < B 형태
- 파라미터:
  - `expr_type`: {덧셈형, 곱셈형}
  - `A`, `B`: 자연수 (결과가 정수 범위가 되도록)
  - 제약: 결과 범위가 자연수로 표현될 것
- 예시(변형 수치): "□ + 28 > 65일 때, □에 들어갈 수 있는 수의 범위를 수직선에 나타내시오."
- 풀이 절차:
  1. 부등식 풀기 → □ > (또는 <, ≥, ≤) 경곗값
  2. '초과/미만/이상/이하' 중 적절한 표현 선택
- 오답 포인트: '초과'와 '이상' 혼동
- 답 형식: fill-blanks (범위 표현)

### 유형 1-6: 체질량지수(BMI) 범위 판별
- 단계: 2단계
- 구조: 체질량지수 = 몸무게(kg) ÷ (키(m) × 키(m))로 계산합니다. 조건(키, 몸무게 범위)이 주어질 때, 체질량지수 구간 판정표를 보고 해당 구간을 구하시오.
- 파라미터:
  - `height_digits`: 키의 십의 자리·일의 자리 범위 조건
  - `weight_digits`: 몸무게 범위 조건
  - `bmi_table`: {저체중: 20 미만, 정상체중: 20~25 미만, 과체중: 25~30 미만, 비만: 30 이상}
  - 제약: 결과 BMI가 정수 또는 간단한 소수
- 예시(변형 수치): "키의 십의 자리 숫자가 1이고 일의 자리 숫자는 4 이상 7 미만인 자연수이며, 몸무게는 60 kg 이상 70 kg 미만일 때 체질량지수 구간을 구하시오."
- 풀이 절차:
  1. 가능한 키·몸무게 범위에서 BMI 최솟값·최댓값 계산
  2. BMI 범위를 판정표와 대조
- 오답 포인트: 나눗셈 순서 혼동, 판정표 경계 이상/이하 오독
- 답 형식: 4지선다(choice) 또는 fill-blanks

### 유형 1-7: 꺾은선그래프 반올림 후 변화 읽기
- 단계: 3단계
- 구조: 연도별 측정값이 표로 주어집니다. 각 값을 반올림하여 천(또는 백)의 자리까지 나타낸 후 꺾은선그래프로 표현할 때, 두 연도 사이 감소(또는 증가)의 크기를 구하시오.
- 파라미터:
  - `years`: 연도 목록 (예: 4~5개)
  - `values`: 각 연도 측정값 (예: 200000~350000 범위)
  - `round_place`: {천, 만, 백}
  - 제약: 반올림 전후로 순위가 바뀌는 연도가 적어도 1개 있을 것
- 예시(변형 수치): "연도별 생산량이 2020년 305607, 2021년 289340, 2022년 312850, 2023년 274215입니다. 반올림하여 만의 자리까지 나타낼 때 가장 큰 감소가 일어난 기간은?"
- 풀이 절차:
  1. 각 값 반올림 적용
  2. 인접 연도 차 계산
  3. 최대 감소 구간 확인
- 오답 포인트: 반올림 전 원래 값으로 계산, 증가와 감소 방향 혼동
- 답 형식: fill-blanks

### 유형 1-8: 복수 어림 조건을 동시 만족하는 수 개수
- 단계: 3단계
- 구조: 다음 조건을 모두 만족하는 수를 찾아 기호를 쓰시오.
  - 조건 1: 올림하여 백의 자리까지 나타내면 A이다.
  - 조건 2: 버림하여 십의 자리까지 나타내면 B이다.
  - 보기: ①~④ (구체적 수치)
- 파라미터:
  - `A`: 올림 후 목표 (예: 4100, 3200 등 백 단위)
  - `B`: 버림 후 목표 (예: 4000, 3190 등 십 단위)
  - `candidates`: 보기 4개 (범위 내 2개 정답, 2개 오답)
  - 제약: A와 B의 조건을 동시 만족하는 범위가 존재할 것
- 예시(변형 수치): "올림하여 백의 자리까지 나타내면 5200이고, 버림하여 십의 자리까지 나타내면 5110인 수를 보기에서 모두 찾으시오. ① 5118 ② 5143 ③ 5102 ④ 5099"
- 풀이 절차:
  1. 올림 조건 역산 → 범위 설정
  2. 버림 조건 역산 → 범위 설정
  3. 두 범위 교집합 → 보기 대입
- 오답 포인트: 범위 교집합 계산 오류, 올림·버림 방향 혼동
- 답 형식: 4지선다(choice, 복수 가능)

## 2. 분수의 곱셈

### 유형 2-1: 범위 부등식 안에 들어갈 자연수 구하기
- 단계: 2단계
- 구조: □ 안에 들어갈 수 있는 자연수를 모두 구하시오. 식: A < □×(p/q) < B (A, B는 자연수 또는 대분수)
- 파라미터:
  - `frac`: 진분수 p/q (기약분수, q ≤ 12)
  - `lower`: 하한 대분수 또는 자연수
  - `upper`: 상한 대분수 또는 자연수
  - 제약: 만족하는 자연수가 3~6개 존재할 것
- 예시(변형 수치): "□ 안에 들어갈 수 있는 자연수를 모두 구하시오. 2(3/5) < □ × (3/7) < 5(1/2)"
- 풀이 절차:
  1. 각 변을 (p/q)로 나눠 □ 범위 결정: lower/(p/q) < □ < upper/(p/q)
  2. 범위를 분수로 계산 → 자연수 추출
- 오답 포인트: 역수 곱하기 실수, 경계값 포함 여부 혼동
- 답 형식: fill-blanks

### 유형 2-2: 속력 × 시간 거리 계산 (대분수 × 대분수)
- 단계: 2단계
- 구조: 어떤 탈것이 한 시간에 A(b/c) km를 달립니다. D시간 E분 동안 달렸을 때 총 거리는 몇 km입니까?
- 파라미터:
  - `speed`: 대분수 속력 (예: 15(1/3) ~ 20(2/5) km/h)
  - `hours`: 시간 정수 부분 (1~3)
  - `minutes`: 분 단위 {15, 20, 30, 40, 45}
  - 제약: 계산 결과가 대분수로 깔끔히 나올 것 (분모 ≤ 20)
- 예시(변형 수치): "자동차가 한 시간에 18(2/5) km를 갑니다. 1시간 40분 동안 달린 거리는 몇 km입니까?"
- 풀이 절차:
  1. 분 → 시간 변환: E분 = E/60 시간 = 기약분수
  2. 총 시간 = hours + 분수
  3. 거리 = speed × 총 시간 (대분수 곱셈)
- 오답 포인트: 분→시간 변환 생략, 대분수를 가분수로 바꾸지 않고 곱셈
- 답 형식: 분수(fraction-input)

### 유형 2-3: 시계 바늘 분수 연산 (하루 낙차 계산)
- 단계: 3단계
- 구조: 어떤 시계는 하루에 A(b/c)분씩 [빠르게/느리게] 갑니다. D일 후 오전 E시를 가리킬 때 실제 시각은 오전 몇 시 몇 분입니까?
- 파라미터:
  - `drift`: 하루 오차 대분수 분 단위 (예: 2(1/3))
  - `direction`: {빠르게, 느리게}
  - `days`: 경과 일수 (5~20)
  - `shown_time`: 시계가 가리키는 시각
  - 제약: 실제 시각이 자정을 넘지 않을 것
- 예시(변형 수치): "어떤 시계는 하루에 1(2/5)분씩 빠르게 갑니다. 8일 후 오전 7시를 가리킬 때 실제 시각은 오전 몇 시 몇 분입니까?"
- 풀이 절차:
  1. 총 누적 오차 = drift × days
  2. 빠른 경우: 실제 시각 = 시계 시각 − 오차
  3. 분→시·분 변환
- 오답 포인트: 오차 방향 혼동(빠른 시계이면 실제는 더 이른 시각), 분→시 변환 오류
- 답 형식: fill-blanks

### 유형 2-4: 전체의 분수만큼 사용 후 남은 양
- 단계: 3단계
- 구조: 총량 N kg의 [재료]가 있습니다. 전체의 A/B는 [용도_1]에 사용하고, 나머지의 C/D는 [용도_2]에 사용하였습니다. 남은 양은 몇 kg입니까?
- 파라미터:
  - `total`: 총량 (대분수 kg, 예: 1(1/3) × 정수)
  - `frac1`: 첫 번째 소비 비율 (진분수)
  - `frac2`: 두 번째 소비 비율 (진분수, 나머지 기준)
  - 제약: 남은 양이 기약분수로 표현될 것
- 예시(변형 수치): "밀가루가 한 봉지에 2(1/4) kg씩 8봉지 있습니다. 전체의 1/3을 빵 만들기에 사용하고, 나머지의 3/5를 과자 만들기에 사용하였습니다. 남은 밀가루는 몇 kg입니까?"
- 풀이 절차:
  1. 총량 = 단위 × 개수
  2. 1차 사용 후 남은 양 = 총량 × (1 − frac1)
  3. 2차 사용 후 남은 양 = 1차 남은 양 × (1 − frac2)
- 오답 포인트: 두 번째 분율이 나머지 기준임을 간과하고 전체 기준으로 계산
- 답 형식: 분수(fraction-input)

### 유형 2-5: 두 사람의 합산 이동 거리
- 단계: 3단계
- 구조: 두 사람이 서로 반대 방향으로 자전거를 탑니다. 가는 속력 A(b/c) km/h, 오는 속력 D(e/f) km/h일 때 T시간 후 두 사람 사이의 거리는 몇 km입니까?
- 파라미터:
  - `speed1`, `speed2`: 두 속력 (대분수)
  - `time`: 경과 시간 (대분수, 예: 1(3/4)시간)
  - 제약: 결과 분모 ≤ 48
- 예시(변형 수치): "현재 지점에서 두 사람이 반대 방향으로 출발합니다. A는 시속 4(3/4) km, B는 시속 3(2/3) km로 1시간 30분 후의 두 사람 간 거리는 몇 km입니까?"
- 풀이 절차:
  1. 각 속력 × 시간 → 각 이동 거리
  2. 두 거리의 합
- 오답 포인트: 한 쪽 거리만 계산, 분 단위 시간 변환 오류
- 답 형식: 분수(fraction-input)

### 유형 2-6: 규칙적 분수 곱 패턴 (연속 곱)
- 단계: 3단계
- 구조: 분자는 홀수가 증가하고 분모도 규칙적으로 증가하는 분수 수열의 첫 항부터 N번째까지의 곱을 구하시오. 예: 1/3 × 3/5 × 5/7 × ⋯ × (2N−1)/(2N+1)
- 파라미터:
  - `pattern_type`: {분자·분모 홀수열, 분자·분모 짝수+홀수 교차}
  - `N`: 항의 수 (50~200)
  - 제약: 연속 분수 소거가 일어나 결과가 단순 분수
- 예시(변형 수치): "1/4 × 4/7 × 7/10 × ⋯ × 148/151 형태로 된 수열의 처음부터 50번째 분수까지의 곱을 구하시오."
- 풀이 절차:
  1. 규칙 파악: n번째 분수의 분자와 분모 표현
  2. 소거 패턴 확인 (분자가 다음 항 분모와 같음)
  3. 결과 = 첫 번째 분자 / 마지막 분모
- 오답 포인트: 소거 패턴 미인식, 항 수 잘못 셈
- 답 형식: 분수(fraction-input)

### 유형 2-7: 넓이 변화 비율 (대분수 곱셈)
- 단계: 3단계
- 구조: 직사각형의 가로를 A(b/c)배 늘이고, 세로를 D(e/f)배 줄였을 때 넓이는 처음의 몇 배입니까?
- 파라미터:
  - `scale_w`: 가로 배율 대분수 (1 초과, 예: 1(1/5))
  - `scale_h`: 세로 배율 대분수 (1 미만, 예: 3/4)
  - 제약: 두 배율의 곱이 기약분수로 결과 ≤ 1
- 예시(변형 수치): "직사각형의 가로를 처음 길이의 1(2/5)배로 늘이고, 세로를 처음 길이의 3/4배로 줄여 새 직사각형을 만들었습니다. 새 직사각형의 넓이는 처음의 몇 배입니까?"
- 풀이 절차:
  1. 넓이 배율 = scale_w × scale_h
  2. 가분수 변환 후 곱셈
- 오답 포인트: 배율을 면적에 직접 더함, 약분 미적용
- 답 형식: 분수(fraction-input)

### 유형 2-8: 합·차 조건으로 두 분수 곱 구하기
- 단계: 3단계
- 구조: 합이 A/B이고 차가 C/D인 두 분수가 있습니다. 두 분수의 곱을 구하시오.
- 파라미터:
  - `sum_frac`: 합 분수 (예: 5/6)
  - `diff_frac`: 차 분수 (예: 2/9)
  - 제약: 두 분수가 각각 진분수이고, 공통분모로 계산 가능할 것; 결과 분수가 기약분수
- 예시(변형 수치): "합이 7/8이고 차가 1/6인 두 분수가 있습니다. 두 분수의 곱을 구하시오."
- 풀이 절차:
  1. 큰 수 = (합 + 차) ÷ 2, 작은 수 = (합 − 차) ÷ 2
  2. 두 수의 곱 계산
- 오답 포인트: 합과 차의 평균을 분수로 계산 시 통분 오류
- 답 형식: 분수(fraction-input)

## 3. 합동과 대칭

> 주의: 이 단원의 대부분 문제는 구체적 도형 그림이 필수입니다.
> 아래 유형 중 그림 없이 텍스트만으로 출제 가능한 것만 수록하였습니다.
> 그림 필수 문제 목록은 하단 "그림 필수 문제 목록" 참조.

### 유형 3-1: 합동 삼각형 각도 추론
- 단계: 2단계, 3단계
- 구조: 삼각형 ①과 삼각형 ②가 서로 합동입니다. 삼각형 ①의 각도 A, B가 주어질 때, 삼각형이 연결·합쳐진 도형에서 특정 각도를 구하시오.
- 파라미터:
  - `angle_a`, `angle_b`: 합동 삼각형의 두 각도 (합이 90° 미만, 예: 35°, 55°)
  - `connection`: 연결 방식 {공유변 연결, 꼭짓점 맞대기}
  - 제약: 구하는 각도가 180° − a − b 형태로 계산 가능
- 예시(변형 수치): "두 삼각형이 서로 합동인 이등변삼각형입니다. 한 각도가 70°이고 다른 각도가 95°일 때, 연결된 도형의 특정 각도 ⓐ를 구하시오."
- 풀이 절차:
  1. 합동 조건으로 대응각 동치 설정
  2. 삼각형 내각의 합 = 180° 이용
  3. 연결 구조에 따라 선형 방정식 세우기
- 오답 포인트: 대응각 잘못 매칭, 이등변삼각형 성질 미적용
- 답 형식: fill-blanks (°)

### 유형 3-2: 선대칭도형 각도 계산
- 단계: 2단계
- 구조: 사각형 또는 삼각형 두 개가 각각 선대칭도형입니다. 주어진 각도를 이용하여 특정 각을 구하시오.
- 파라미터:
  - `given_angle`: 주어진 각도 (예: 144°)
  - `shape_type`: {삼각형, 사각형}
  - 제약: 선대칭 성질(대응각 동치)과 다각형 내각 합만으로 풀릴 것
- 예시(변형 수치): "사각형 ①이 선대칭도형이고 삼각형 ②가 선대칭도형입니다. 각도 X = 136°일 때, 삼각형 내의 각도 Y를 구하시오."
- 풀이 절차:
  1. 선대칭 → 대응각 같음 적용
  2. 사각형 내각 합 360° 또는 삼각형 180° 이용
  3. 목표 각도 계산
- 오답 포인트: 선대칭 조건 미적용, 내각 합 공식 혼동
- 답 형식: fill-blanks (°)

### 유형 3-3: 지도 기호 중 선대칭·점대칭 동시 해당 개수
- 단계: 2단계
- 구조: 제시된 N개의 기호 그림 중 선대칭도형이면서 동시에 점대칭도형인 것의 개수를 구하시오.
- 파라미터:
  - `symbols`: 기호 집합 (예: 8~12개 간단한 기호)
  - 제약: 선대칭만, 점대칭만, 둘 다, 둘 다 아닌 것이 각각 1개 이상 포함
- 예시(변형 수치): "다음 기호 중 선대칭도형이면서 점대칭도형인 것은 모두 몇 개입니까? 기호 목록: ✚, ☆, ◇, ▲, ●, ⊕"
- 풀이 절차:
  1. 각 기호에 대해 선대칭 여부 판단
  2. 각 기호에 대해 점대칭 여부 판단
  3. 교집합 개수 세기
- 오답 포인트: 선대칭과 점대칭 정의 혼동, 정사각형·원 등을 점대칭에서 누락
- 답 형식: 자연수(fill-blanks)

### 유형 3-4: 정사각형 종이 접기 각도
- 단계: 2단계, 3단계
- 구조: 정사각형 모양의 종이를 [접기 방법]으로 접었을 때 두 각도 ①과 ②의 차를 구하시오. (각도 하나가 주어짐)
- 파라미터:
  - `fold_angle`: 접힌 부분의 각도 (예: 32°)
  - 제약: 풀이에 합동 삼각형 성질과 직각 성질만 사용
- 예시(변형 수치): "정사각형 종이를 접었을 때 생긴 각도가 38°입니다. 나머지 두 각도 ①과 ②의 차를 구하시오."
- 풀이 절차:
  1. 접기 → 합동 삼각형 생성
  2. 대응각 동치 적용
  3. 직각(90°) 조건 이용해 미지각 계산
  4. 두 각도의 차
- 오답 포인트: 접힌 삼각형의 합동 관계 미인식, 직각 조건 누락
- 답 형식: fill-blanks (°)

### 유형 3-5: 수 카드로 만드는 큰 정대칭수
- 단계: 3단계
- 구조: 주어진 숫자 카드 N장을 사용하여 [n자리] 정대칭수(각 자릿수와 대응 자릿수가 같은 수)를 만들 때 가장 크거나 가장 작은 수를 구하시오.
- 파라미터:
  - `cards`: 숫자 카드 집합 (예: {0,1,2,5,6,9})
  - `digits`: 만들 수의 자릿수 (4~6)
  - `target`: {가장 큰 수, 가장 작은 수}
  - 제약: 첫 자리 0 불가, 카드 재사용 조건 명시
- 예시(변형 수치): "숫자 카드 {0,1,2,5,6,9}를 같은 카드를 여러 번 사용하여 4자리 팰린드롬 수를 만들 때, 6□□9보다 큰 팰린드롬 수는 모두 몇 개입니까?"
- 풀이 절차:
  1. 팰린드롬 구조 파악 (n자리 → 앞 절반으로 결정)
  2. 조건 범위 안에서 경우 열거
- 오답 포인트: 팰린드롬 자릿수 대응 오류, 첫 자리 0 포함 실수
- 답 형식: 자연수(fill-blanks)

## 4. 소수의 곱셈

### 유형 4-1: 부등식 범위 안의 자연수 합
- 단계: 2단계
- 구조: A × B < □ < C × D에서 □ 안에 들어갈 수 있는 가장 큰 자연수와 가장 작은 자연수의 합을 구하시오.
- 파라미터:
  - `A`, `B`: 소수 두 자리 × 자연수 또는 소수 (예: 1.47 × 19)
  - `C`, `D`: 자연수 × 소수 (예: 46 × 0.835)
  - 제약: 두 계산값이 10 이상 차이 나서 자연수가 다수 존재
- 예시(변형 수치): "1.63 × 21 < □ < 53 × 0.74에서 □에 들어갈 수 있는 가장 큰 자연수와 가장 작은 자연수의 합을 구하시오."
- 풀이 절차:
  1. A×B, C×D 각각 계산
  2. 범위 내 자연수 최대·최솟값 확인 (초과·미만 범위)
  3. 두 값의 합
- 오답 포인트: 부등호 방향(초과, 미만) 경계 포함 오류
- 답 형식: 자연수(fill-blanks)

### 유형 4-2: 연립 비율 방정식 (가장 큰 수는 가장 작은 수의 몇 배)
- 단계: 2단계
- 구조: 세 식이 주어집니다.
  ① A × □ = B (소수)
  ② □ × C = D (소수)
  ③ E × □ = F (소수)
  각 □ 값을 구하고, 가장 큰 □는 가장 작은 □의 몇 배인지 구하시오.
- 파라미터:
  - 각 식은 (자연수 × 소수 = 소수) 또는 (소수 × 소수 = 소수) 형태
  - 제약: 세 □ 값이 {0.1, 0.01, 0.001, 10, 100} 중 서로 다른 값
- 예시(변형 수치): "① 240 × □ = 24, ② □ × 3.17 = 317, ③ 83 × □ = 0.83 에서 가장 큰 □는 가장 작은 □의 몇 배입니까?"
- 풀이 절차:
  1. 각 □ = 결과 ÷ 앞 수 계산
  2. 세 값 비교 → 최대 / 최소
- 오답 포인트: 소수점 위치 이동 방향 오류
- 답 형식: 자연수(fill-blanks)

### 유형 4-3: 소수 곱 카드 배열 최댓값 최솟값
- 단계: 2단계
- 구조: 수 카드 N장을 한 번씩 사용하여 (소수 두 자리 수) × (소수 한 자리 수) 형태의 식을 만들 때, 계산 결과가 가장 클/작을 때의 곱을 구하시오.
- 파라미터:
  - `cards`: 1~9 중 5개 (0 포함 가능, 단 소수 첫째 자리 0 불가)
  - `target`: {최댓값, 최솟값}
  - 제약: 카드 재사용 불가, 소수 형식 강제
- 예시(변형 수치): "수 카드 {1, 3, 5, 7, 8}을 한 번씩 사용하여 (소수 두 자리 수) × (소수 한 자리 수)를 만들 때 곱이 가장 클 때의 값을 구하시오."
- 풀이 절차:
  1. 큰 수를 높은 자릿값에 배치하는 전략 적용
  2. 가능한 배치 비교 (3~4가지)
  3. 최댓값/최솟값 확인
- 오답 포인트: 자릿값과 실제 곱 크기의 직관적 혼동
- 답 형식: 소수(decimal-input)

### 유형 4-4: 반복 곱 후 소수 자릿수 패턴
- 단계: 2단계
- 구조: 어떤 소수 D를 N번 곱할 때, 소수 M번째 자리 숫자를 구하시오. (반복 주기 파악)
- 파라미터:
  - `D`: 소수 (예: 0.7, 0.3, 0.8)
  - `N`: 반복 횟수 (20~50)
  - `M_place`: 구하는 소수 자릿수 (25~50번째)
  - 제약: 자릿수 패턴 주기가 2~5
- 예시(변형 수치): "0.6을 40번 곱할 때의 소수 40번째 자리 숫자를 구하시오."
- 풀이 절차:
  1. D^1, D^2, D^3 ... 계산하여 해당 자릿수 추출
  2. 주기 확인
  3. N mod 주기로 나머지 계산 → 해당 자릿수
- 오답 포인트: 주기 착오, mod 나머지 0 처리 실수
- 답 형식: 자연수(fill-blanks)

### 유형 4-5: 약품·화학 농도 또는 비율 곱셈 응용
- 단계: 3단계
- 구조: (기준값) × (비율 소수) = 결과를 이용하는 실생활 문제. 두 단계 비율 계산: 올해 저금액 = 작년 × A배, 목표액 = 작년 × B배, 차액을 구하시오.
- 파라미터:
  - `base`: 작년 기준값 (예: 75000~120000원)
  - `ratio1`: 현재 비율 소수 (예: 0.82)
  - `ratio2`: 목표 배율 소수 (예: 1.5)
  - 제약: 두 계산 모두 정수 결과
- 예시(변형 수치): "작년 저금액이 92000원일 때, 올해는 작년의 0.78배를 저금했습니다. 올해 목표 저금액은 작년의 1.4배일 때 목표를 채우려면 얼마를 더 저금해야 합니까?"
- 풀이 절차:
  1. 현재 저금액 = base × ratio1
  2. 목표 저금액 = base × ratio2
  3. 차액 = 목표 − 현재
- 오답 포인트: 배율 방향 혼동, 소수 곱셈 소수점 오류
- 답 형식: 자연수(fill-blanks)

### 유형 4-6: 겹쳐지는 테이프 전체 길이
- 단계: 3단계
- 구조: 길이 A cm인 테이프 N장을 B cm씩 겹쳐서 일렬로 이어 붙였습니다. 전체 길이는 몇 cm입니까?
- 파라미터:
  - `tape_length`: 테이프 한 장 길이 소수 (예: 18.6)
  - `count`: 장수 (10~50)
  - `overlap`: 겹치는 길이 소수 (예: 3.14)
  - 제약: 겹치는 부분 = count − 1 개소
- 예시(변형 수치): "길이가 24.5 cm인 색 테이프 18장을 4.32 cm씩 겹쳐서 한 줄로 이어 붙였습니다. 전체 길이는 몇 cm입니까?"
- 풀이 절차:
  1. 전체 합 = tape_length × count
  2. 겹친 부분 합 = overlap × (count − 1)
  3. 전체 길이 = 전체 합 − 겹친 부분 합
- 오답 포인트: 겹치는 부분 개수를 count로 잘못 셈
- 답 형식: 소수(decimal-input)

### 유형 4-7: 열차·자동차 터널 통과 시간·거리
- 단계: 3단계
- 구조: 길이 A km인 터널을 분당 B km 속력으로 완전히 통과하는 데 C분이 걸렸습니다. 열차의 길이는 몇 m입니까?
- 파라미터:
  - `tunnel_length`: 터널 길이 소수 (km, 예: 1.8)
  - `speed`: 분당 속력 소수 (km/min, 예: 0.32)
  - `time`: 통과 시간 분 (예: 6(1/2))
  - 제약: 열차 길이 = speed × time − tunnel_length, 결과 m 단위 정수
- 예시(변형 수치): "일정한 빠르기로 1분에 2.4 km를 달리는 기차가 길이 2.7 km인 터널을 완전히 통과하는 데 2분 30초가 걸렸습니다. 기차의 길이는 몇 m입니까?"
- 풀이 절차:
  1. 통과 시간 분수 변환
  2. 이동 거리 = speed × time
  3. 기차 길이 = 이동 거리 − 터널 길이 (km → m 변환)
- 오답 포인트: 통과 거리 = 터널만으로 오해, km↔m 변환 오류
- 답 형식: 자연수(fill-blanks)

### 유형 4-8: 기온 변화율 (소수 곱셈 연쇄)
- 단계: 3단계
- 구조: 높이가 H m 높아질 때마다 기온이 D°C씩 내려갑니다. 지면 기온이 T°C이고 높이 H_target m 지점의 기온을 구하시오.
- 파라미터:
  - `rate`: 100m 당 기온 하강률 소수 (예: 0.6°C)
  - `base_temp`: 지면 기온 소수 (예: 24.5)
  - `height`: 목표 높이 (예: 1500)
  - 제약: 결과가 소수 둘째 자리까지
- 예시(변형 수치): "높이가 100 m 높아질 때마다 기온이 0.5°C씩 내려갑니다. 지면 기온이 22.8°C일 때 1200 m 지점의 기온은 몇 °C입니까?"
- 풀이 절차:
  1. 높이 단위 수 = height ÷ 100
  2. 총 하강 = rate × 단위 수
  3. 지점 기온 = base_temp − 총 하강
- 오답 포인트: 단위 환산 오류, 빼기 방향 오류
- 답 형식: 소수(decimal-input)

## 5. 직육면체

### 유형 5-1: 직육면체 모서리 합 구하기
- 단계: 2단계
- 구조: 직육면체 ①의 모든 모서리의 길이가 같습니다. 직육면체의 한 면의 모서리의 길이 합(가로 a, 세로 b, 높이 c)이 주어질 때 전체 모서리의 길이의 합을 구하시오.
- 파라미터:
  - `a`, `b`, `c`: 세 모서리 길이 cm 자연수 (예: 6, 8, 13)
  - 제약: 모서리 합 = 4(a + b + c)
- 예시(변형 수치): "직육면체의 가로 7 cm, 세로 5 cm, 높이 11 cm일 때 모든 모서리의 길이의 합은 몇 cm입니까?"
- 풀이 절차: 4 × (a + b + c) 계산
- 오답 포인트: 모서리 개수 12개를 4묶음으로 보지 않고 임의 계산
- 답 형식: 자연수(fill-blanks)

### 유형 5-2: 두 직육면체 조합 상자 모서리 합
- 단계: 2단계
- 구조: 모양과 크기가 같은 두 직육면체를 이어 붙여 큰 직육면체를 만들 때, 모든 모서리의 합을 구하시오. (두 방향으로 붙이는 경우 각각 계산)
- 파라미터:
  - `a`, `b`, `c`: 작은 직육면체 세 모서리 (예: 15 cm, 12 cm, 20 cm)
  - `combine_axis`: 붙이는 방향 {가로, 세로, 높이}
  - 제약: 이어 붙인 방향의 모서리 길이가 2배
- 예시(변형 수치): "가로 10 cm, 세로 8 cm, 높이 14 cm인 직육면체 2개를 세로 방향으로 이어 붙였을 때 모서리 합을 구하시오."
- 풀이 절차:
  1. 붙이는 축 길이 × 2 → 새 직육면체 세 변
  2. 모서리 합 = 4(a' + b' + c')
- 오답 포인트: 합쳐진 직육면체의 세 변 잘못 설정
- 답 형식: 자연수(fill-blanks)

### 유형 5-3: 정육면체 칠하기 후 자르기 — 한 면만 색칠된 개수
- 단계: 2단계
- 구조: 한 모서리 길이 A cm인 정육면체를 겉면 모두 색칠한 후 한 모서리 길이 B cm인 정육면체로 잘랐을 때, 한 면만 색칠된 작은 정육면체는 몇 개입니까?
- 파라미터:
  - `big`: 큰 정육면체 모서리 (예: 12, 15)
  - `small`: 작은 정육면체 모서리 (예: 3, 5)
  - 제약: big이 small의 배수
  - `paint`: {전면 칠, 밑면 포함 전면 칠}
- 예시(변형 수치): "한 모서리 길이 15 cm인 정육면체를 겉면 모두 색칠한 후 한 모서리 5 cm인 정육면체로 잘랐습니다. 한 면만 색칠된 것은 몇 개입니까?"
- 풀이 절차:
  1. n = big ÷ small (한 방향 분할 수)
  2. 표면에 있는 작은 정육면체: 각 면의 중앙부 = (n−2)² 개 × 6면
  3. 단, 모서리 포함 여부에 따라 조정
- 오답 포인트: 꼭짓점·모서리 중복 계산
- 답 형식: 자연수(fill-blanks)

### 유형 5-4: 전개도에서 모서리 길이 추론
- 단계: 2단계
- 구조: 직육면체의 전개도가 주어지고 일부 치수가 표기되어 있습니다. 나머지 변의 길이를 구하시오.

  - 제약: 전개도에서 맞닿는 면 관계로 나머지 변 유추 가능
- 예시(변형 수치): "직육면체 전개도에서 일부 변의 길이가 24 cm, 5 cm, 15 cm로 주어져 있습니다. 특정 변의 길이를 구하시오."

  1. 전개도 면 연결 관계 파악
  2. 서로 마주 보는 면은 같은 크기임을 이용
  3. 미지 변 계산
- 오답 포인트: 마주 보는 면과 붙어 있는 면 혼동
- 답 형식: 자연수(fill-blanks)

### 유형 5-5: 직육면체 쌓기 최솟값·최댓값 모서리

- 구조: 쌓기나무로 정사각형 모양의 상자를 만들 때 사용된 쌓기나무 최솟값/최댓값을 구하시오.

  - `target`: {최소 개수, 최대 개수}

### 유형 5-6: 전개도 접기 — 마주 보는 면 수 곱

  - `net_shape`: 전개도 형태 기술 (십자형, ㄴ자형 등)

- 예시(변형 수치): "다음 전개도를 접었을 때 마주 보는 면의 수의 곱 중 가장 큰 것을 구하시오. 면의 숫자: 3, 5, 6, 8, 11, 14"
- 풀이 절차:
  1. 전개도에서 마주 보는 면 3쌍 파악
  2. 각 쌍의 곱 계산
  3. 최댓값 선택
- 오답 포인트: 인접한 면을 마주 보는 면으로 오인
- 답 형식: 자연수(fill-blanks)

### 유형 5-7: 정육면체 쌓기 — 겉면 칠하기 후 층별 개수
- 단계: 3단계
- 구조: 한 모서리 A cm 정육면체를 모든 겉면에 색칠하고, 한 모서리 1 cm인 정육면체로 잘랐을 때 정확히 두 면이 색칠된 것은 모두 몇 개입니까?
- 파라미터:
  - `n`: 한 방향 분할 수 (예: 4, 5)
  - `paint_faces`: {6면 모두, 5면(바닥 제외)}
  - `target_paint_count`: {1면, 2면, 3면}
- 예시(변형 수치): "한 모서리 5 cm인 정육면체를 바닥 포함 모든 겉면에 색칠한 후 한 모서리 1 cm로 잘랐을 때, 정확히 두 면에 색칠된 것은 몇 개입니까?"
- 풀이 절차:
  1. n = A ÷ 1 = A
  2. 모서리에 위치한 정육면체: 12 × (n−2)개 (3면 칠)
  3. 면에 위치한 정육면체: 6 × (n−2)² (1면 칠)
  4. 두 면 = 모서리 위치 (단, 꼭짓점 제외)
- 오답 포인트: 꼭짓점(3면) vs 모서리(2면) vs 면 중앙(1면) 구분 오류
- 답 형식: 자연수(fill-blanks)

## 6. 평균과 가능성

### 유형 6-1: 전체 합 역산으로 평균 구하기
- 단계: 2단계
- 구조: N개 자료의 평균이 A일 때, 특정 자료 하나를 바꾸거나 추가·제거한 후의 평균을 구하시오. 또는 평균이 주어졌을 때 빠진 자료의 값을 구하시오.
- 파라미터:
  - `n`: 자료 수 (4~8)
  - `avg`: 주어진 평균 (자연수 또는 소수 한 자리)
  - `known_values`: 이미 알려진 n−1개 값
  - 제약: 미지 값이 자연수로 나올 것
- 예시(변형 수치): "다섯 과목 점수의 평균이 86점입니다. 국어 92, 사회 88, 과학 79, 영어 90점일 때 수학 점수는 몇 점입니까?"
- 풀이 절차:
  1. 전체 합 = avg × n
  2. 미지 값 = 전체 합 − 알려진 값들의 합
- 오답 포인트: n을 잘못 셈, 전체 합 대신 평균끼리 더함
- 답 형식: 자연수(fill-blanks)

### 유형 6-2: 두 집단 합산 평균 비교
- 단계: 2단계
- 구조: 두 모둠의 자료가 각각 주어집니다. 두 모둠을 합쳤을 때의 평균보다 3 이상 많은 자료는 몇 개입니까?
- 파라미터:
  - `group_a`: 모둠 A 값 목록 (4~6개)
  - `group_b`: 모둠 B 값 목록 (4~6개)
  - `threshold_diff`: 기준 차이 (예: 3)
  - 제약: 두 모둠 크기가 달라 단순 평균의 평균과 다를 것
- 예시(변형 수치): "모둠 A의 읽은 책 수: 6, 8, 5, 9, 3 / 모둠 B: 7, 10, 4, 8입니다. 두 모둠 전체 평균보다 4권 이상 많이 읽은 사람은 몇 명입니까?"
- 풀이 절차:
  1. 전체 합 = A합 + B합, 전체 n = |A| + |B|
  2. 전체 평균 계산
  3. 기준 = 전체 평균 + threshold_diff
  4. 각 자료와 비교하여 개수 계산
- 오답 포인트: 각 모둠 평균의 평균으로 계산 (크기가 다를 때 오류)
- 답 형식: 자연수(fill-blanks)

### 유형 6-3: 가능성 표현 — 일어날 가능성 비교
- 단계: 2단계
- 구조: 주머니에 N개의 공이 들어 있습니다. 특정 색 공을 꺼낼 가능성을 수로 표현하시오. 또는 두 사건의 가능성을 비교하시오.
- 파라미터:
  - `total`: 전체 공 수 (예: 10~20)
  - `target_count`: 특정 색 공 수
  - 제약: 가능성이 0, 1, 1/2, 또는 간단한 분수
- 예시(변형 수치): "빨간 구슬 4개, 파란 구슬 6개가 들어 있는 주머니에서 구슬 한 개를 꺼낼 때, 빨간 구슬이 나올 가능성을 수로 나타내시오."
- 풀이 절차:
  1. 가능성 = target_count ÷ total
  2. 기약분수로 표현
- 오답 포인트: 분모를 전체가 아닌 나머지로 설정
- 답 형식: 분수(fraction-input)

### 유형 6-4: 복수 조건의 가능성 계산
- 단계: 2단계
- 구조: N장의 수 카드에서 한 장 뽑을 때 조건을 만족할 가능성을 수로 나타내시오. (조건: 특정 배수이거나, 소수이거나, 범위 조건)
- 파라미터:
  - `range`: 카드 범위 (예: 1~20)
  - `condition`: {k의 배수, 홀수, 소수, A 이상 B 이하}
  - 제약: 결과 분수가 기약분수
- 예시(변형 수치): "1부터 20까지 적힌 카드 중 한 장을 뽑을 때, 3의 배수가 나올 가능성을 수로 나타내시오."
- 풀이 절차:
  1. 범위 내 조건 만족 수 열거
  2. 가능성 = 개수 ÷ 전체
  3. 기약분수 변환
- 오답 포인트: 범위 경계 포함 여부, 배수 열거 오류
- 답 형식: 분수(fraction-input)

### 유형 6-5: 평균 역산 — 점수 조작 후 평균
- 단계: 2단계
- 구조: 총 N명의 점수 자료 중 상위 K개를 제외한 나머지의 평균을 구하시오. 또는 특정 구간의 평균과 전체 평균의 차를 구하시오.
- 파라미터:
  - `scores`: 점수 목록 또는 표 (N = 4~8)
  - `exclude_count`: 제외할 상위/하위 개수
  - 제약: 제외 후 평균이 자연수
- 예시(변형 수치): "6회까지의 단원평가 점수: 92, 85, 80, 96, 78, 90. 가장 높은 점수 2개를 제외한 나머지의 평균은 몇 점입니까?"
- 풀이 절차:
  1. 상위 K개 제거
  2. 나머지 합 ÷ (N − K)
- 오답 포인트: 제외 후 개수 계산 오류
- 답 형식: 자연수(fill-blanks)

### 유형 6-6: 넓이당 인구밀도·생산량 평균 비교
- 단계: 3단계
- 구조: 표에 지역별 특정 값이 주어집니다. 전체 평균보다 낮은(또는 높은) 지역의 수를 구하시오.
- 파라미터:
  - `regions`: 지역 이름 목록 (4~6개)
  - `values`: 각 지역의 수치
  - `condition`: {평균 이상, 평균 초과, 평균보다 A 이상 낮음}
  - 제약: 평균이 자연수 또는 소수 한 자리
- 예시(변형 수치): "5개 마을의 하루 생산량(단위 kg): 가 150, 나 220, 다 180, 라 95, 마 310. 전체 평균보다 낮은 마을은 몇 개입니까?"
- 풀이 절차:
  1. 전체 합 ÷ N = 평균
  2. 각 값과 비교
  3. 조건 만족 개수 계산
- 오답 포인트: 중앙값과 평균 혼동
- 답 형식: 자연수(fill-blanks)

### 유형 6-7: 신입 회원 가입 후 평균 변화
- 단계: 3단계
- 구조: 기존 N명의 평균이 A이고, 새로 M명이 가입합니다. 새 회원의 값이 주어질 때 전체 평균을 구하시오. 반대로, 전체 평균이 주어질 때 새 회원의 값을 구하시오.
- 파라미터:
  - `n_old`: 기존 인원
  - `avg_old`: 기존 평균
  - `n_new`: 새 인원 수
  - `new_values` 또는 `new_avg`: 새 회원 값 (또는 목표 전체 평균)
  - 제약: 결과가 자연수
- 예시(변형 수치): "32명 동호회의 평균 나이가 43세입니다. 6명이 새로 가입하여 전체 평균이 41세가 되었을 때, 새로 가입한 회원들의 평균 나이는 몇 세입니까?"
- 풀이 절차:
  1. 기존 합 = n_old × avg_old
  2. 전체 합 = (n_old + n_new) × 목표 평균
  3. 새 회원 합 = 전체 합 − 기존 합
  4. 새 회원 평균 = 새 회원 합 ÷ n_new
- 오답 포인트: 역산 순서 오류, 전체 인원수 잘못 셈
- 답 형식: 자연수(fill-blanks)

### 유형 6-8: 최고 기록 제외 평균 비교
- 단계: 3단계
- 구조: 전체 평균과 특정 기록 제외 후 평균이 주어집니다. 제외된 기록의 값을 구하시오.
- 파라미터:
  - `n`: 전체 자료 수
  - `avg_all`: 전체 평균
  - `n_excl`: 제외 후 자료 수 (= n − 1)
  - `avg_excl`: 제외 후 평균
  - 제약: 결과가 자연수
- 예시(변형 수치): "오디션 참가자 5명의 전체 점수 평균이 71.5점이고, 가장 높은 점수를 제외한 4명의 평균이 66점일 때, 제외된 점수는 몇 점입니까?"
- 풀이 절차:
  1. 전체 합 = avg_all × n
  2. 제외 후 합 = avg_excl × n_excl
  3. 제외된 값 = 전체 합 − 제외 후 합
- 오답 포인트: 전체 합 계산 시 인원 혼동
- 답 형식: 자연수(fill-blanks)

## 그림 필수 문제 목록 (생성기 제외 대상)

아래 문제들은 구체적 도형·전개도·그림 없이는 풀 수 없어 텍스트 생성기에서 제외합니다.

### 단원 3. 합동과 대칭 — 거의 전체
- 합동 삼각형 넓이 계산 (두 삼각형 합동 + 치수 그림 필수)
- 선대칭도형 돌레 계산 (도형 그림 + 치수 필수)
- 정삼각형 결합 합동 쌍 개수 (삼각형 분할 그림 필수)
- 점대칭도형 완성 후 돌레 (격자 그림 + 치수 필수)
- 정사각형 종이 접어 꼭짓점 연결 (접기 과정 그림 필수)
- 선대칭도형 직선 위 이동 넓이 차 (도형 그림 필수)
- 두 정사각형 대칭 완성 후 겹치는 부분 (도형 그림 필수)
- 직사각형 선대칭 완성 후 넓이 (전개 과정 그림 필수)

### 단원 5. 직육면체 — 일부
- 전개도 접기 → 마주 보는 면 기호 매칭 (전개도 그림 필수)
- 직육면체 여러 방향 뷰 → □ 치수 맞추기 (앞·위·옆 뷰 그림 필수)
- 쌓기나무 정육면체 쌓기 소마큐브 조각 숫자 (3D 그림 필수)
- 정육면체 끈 감기 전개도에 끈 경로 그리기 (전개도 그림 필수)
- 층별 쌓기 나무 격자 기록표 완성 (격자 그림 필수)
- 빈틈없이 쌓인 직육면체 색칠 면 찾기 (3D 배열 그림 필수)
