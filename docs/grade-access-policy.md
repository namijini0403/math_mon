# 학년별 콘텐츠 접근 정책 (사용자 확정 2026-06-13)

학교 공식 앱으로서 "복습은 권장, 선행은 비권장(차단은 아님)"을 게임 경제로 구현한다.
운영: 담임이 인증(반)코드를 발급 → 코드에 학년-학기를 심어 학생의 "현재 학기"를 정한다.

## 영역별 동작

| 영역 | 판정 | 진입 시 | 보상 |
|---|---|---|---|
| **본인 학년-학기** (디폴트) | 코드의 학년-학기와 일치 | 바로 이용 | 정상 (XP·별·카드·배지 전부) |
| **이전 영역 (복습)** | 학생 현재 학기보다 과거 | 안내 모달 "복습하려고요? 복습 중요하죠!!" | **시간여행 배지만.** XP·카드·별 미지급 |
| **선행 영역** | 학생 현재 학기보다 미래 | 경고 모달 "선행에 도전할 순 있지만 경험치는 오르지 않아요" | **전무.** XP·카드·배지·별 모두 0 |

### 세부 규칙
- **한 학기 내 단원은 순서 무관 자유 이용** (반마다 진도·단원 순서가 다름) — 기존 "단원 잠금은 단원별 독립" 유지.
- **복습 보상을 배지만으로 제한하는 이유**: 쉬운 이전 학년만 반복해 XP를 파밍하는 어뷰징 방지.
  복습 자체는 막지 않고 권장하되, 진행 보상(XP/별)은 현재 학기에서만.
- **시간여행 배지**: 복습 영역을 클리어하면 받는 신규 배지(예: `time-traveler`, 누적형 `time-traveler-N`).
  코드 렌더 SVG 엠블럼(BadgeEmblem)으로 — 모래시계/나침반 글리프 + violet 계열.
- **선행은 "구경·도전"만 가능**: 문제는 풀리되 결과 화면에 보상 0 + "아직 학교에서 배우지 않은 내용이에요" 안내.
- **학년 정보 미설정(코드에 학년 없음) 폴백**: 기존처럼 전체 열림(미인증·체험 계정 호환).

## 인증(반)코드 형식 (확정)
- **코드 끝 2자리 = 학년 + 학기** (예: `STAR52` → 5학년 2학기, `TIGER31` → 3학년 1학기).
  앞부분은 반 이름 자유, 마지막 2글자가 [1-6][1-2].
- 파싱: 코드 끝이 `[1-6][1-2]` 패턴이면 → `g{학년}s{학기}`. 아니면 미설정(전체 열림 폴백).
- 학기가 바뀌면 교사가 코드 끝자리만 바꿔 재발급(예: 학년말 …52 → 신학기 …61).

## 구현 항목 (착수 시)
1. **학년 정보 수집**: 위 형식으로 반코드 파싱 → `store`에 `gradeSemester`(예: 'g5s2') 저장.
   (현재 로그인은 닉네임·반코드·비번·기기만 받아 학년을 모름.)
2. **영역 판정 유틸**: 스테이지/학기 id(g1s1~g6s2)와 학생 현재 학기를 비교해 `'current' | 'review' | 'ahead'` 반환.
3. **진입 가드 + 모달**: 레슨/스테이지 진입 시 영역별 안내·경고 모달.
4. **보상 분기**: LessonPage 클리어 보상 계산에서 영역에 따라 XP/카드/별/배지 게이팅.
   - current=정상, review=시간여행 배지만, ahead=0.
5. **시간여행 배지** 추가 (badges.ts BADGES + BadgeStats에 reviewCleared 등).
6. **UI**: 홈 학기 탭에서 현재 학기를 디폴트·강조, 이전=「복습」, 미래=「선행 도전」 라벨.

## 구현 상태 (2026-06-13)
- ✅ 반코드 파싱(gradeAccess.ts) · 영역 판정(current/review/ahead) · 진입 모달 · 보상 게이팅(LessonPage) · 시간여행 배지(time-traveler, time-traveler-20) · 홈 학기탭 기본=학생 학기·"현재" 마커·복습/선행 배너(UnitMapPage).
- 복습/선행은 XP·별·카드·격파메달·콤보배지 전부 미지급. 복습만 recordReviewClear→시간여행 배지.
- 학생 학기는 classCode에서 파생(별도 persist 필드 없음). 미설정 시 전체 'current' 폴백.

## 후속 적용 — ✅ 완료 (2026-06-13)
- **연습(PracticePage)·단원평가(ExamPage)·총괄평가(FinalExamPage)** 모두 동일 영역 판정으로 보상 게이팅 적용:
  - 연습: 복습/선행은 XP·드래곤 세트 적립 안 함(skillStats 정확도는 기록 — 문제 선택용). 상단 안내 배너.
  - 단원평가: 복습/선행은 XP·메달·히든미션카드 미지급, 결과에 안내.
  - 총괄평가: 복습/선행은 XP·보상카드·배지 미지급, 결과에 안내.
- 공용 헬퍼 `accessZoneForSemester`(학기 기반)·`accessZone`(unit 기반) + 단위테스트(gradeAccess.test.ts).

## 미결정
- 별(최고기록) 표시 정책: 복습/선행은 미기록이 기본(파밍 방지 일관성) — 진행감과 trade-off, 조정 가능.
